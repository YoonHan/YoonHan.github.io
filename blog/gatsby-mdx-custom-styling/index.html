Gatsby MDX 에서 렌더링된 태그 custom styling 하기

February 17, 2022

본 블로그는 Gatsby 를 사용하여 구성했다.
이후 입맛에 맞게 CSS를 정의하여 사용 중이었는데, 전역 CSS로 스타일링 하려니 여러 CSS 파일 간 충돌 문제가 발생했다.(당연한 것이지만..)
마침 공식 홈페이지에 해결 가이드가 있어서 해당 내용을 정리해 보았다.

기존의 상태

다음은 {mdx.slug}.js 의 파일 내용이다. blogPost.module.css 안의 스타일을 그대로 불러와 사용했었다. 이 파일 안에 정의된 스타일에 따라 <MDXRenderer> 컴포넌트 안에 렌더링 되는 태그들의 스타일이 정해졌다.

/* {mdx.slug}.js */ import * as React from 'react' import { MDXRenderer } from 'gatsby-plugin-mdx' // styles import './blogPost.module.css' (...) // Component const BlogPost = ({ data }) => { (...) return ( <div className="blog-post" style={blogPostStyles}> (...) <div style={MarkdownWrapperStyle}> <MDXRenderer> { data.mdx.body } </MDXRenderer> </div> </div> ) }

바뀐 뒤의 상태

<MDXProvider> 에 제공할 커스텀 컴포넌트들을 관리할 파일을 새로 생성(MDXDesignSystem.js)하였다.
아래는 각각 {mdx.slug}.js, MDXDesignSystem.js 파일의 내용이다.

/* {mdx.slug}.js */ // blog post design system import MDXCustomStyles from '../../components/MDXDesignSystem' (...) const BlogPost = ({ data }) => { (...) return ( <div className="blog-post" style={blogPostStyles}> (...) <div style={MarkdownWrapperStyle}> {/* MDXProvider 로 감싼 뒤 커스텀 컴포넌트를 전달 */} <MDXProvider components={MDXCustomStyles} > <MDXRenderer> { data.mdx.body } </MDXRenderer> </MDXProvider> </div> </div> ) } /* MDXDesignSystem.js */ import * as React from 'react' (...) // styles const h2Style = { marginTop: '2rem', position: 'relative', } (...) // custom tag components export function H2(props) { return ( <h2 style={h2Style}> <div style={labelStyle}></div> {props.children} </h2> ) } (...) // 커스텀 컴포넌트 매핑 객체 const MDXCustomStyles= { h2: H2, (...) } export default MDXCustomStyles

기존 <MDXRenderer> 컴포넌트를 <MDXProvider> 로 감싸 주었고,


<MDXProvider> 컴포넌트의 components prop에

// javascript 객체 { [커스터마이징을 원하는 태그]: [그 태그와 mapping 될 react component] }

의 형식을 가진 객체를 전달했다.
그랬더니 <MDXRenderer> 컴포넌트의 렌더링 결과물에 커스텀 스타일이 적용되었다.
